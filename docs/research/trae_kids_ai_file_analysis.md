## 核心主题提取

### 主题 1：产品定位与价值主张
- 项目名称：安心童伴 AI（备选：童伴守护 Agent、小树洞 AI、萤火童伴、童心盾、小鹿守护 AI）
- 目标用户：8-12 岁儿童（主要使用者）+ 家长（决策者）
- 核心价值：让孩子获得安全 AI 陪伴，让家长获得适度守护能力
- 赛道定位：生活娱乐 + 社会公益（双赛道策略）

### 主题 2：功能架构（双端闭环）
- **儿童端**：安全聊天、故事陪伴、儿童百科问答、情绪树洞、安全教育小剧场
- **家长端**：今日对话摘要、风险事件提醒、使用时长统计、话题偏好设置、年龄设置

### 主题 3：多层安全 Harness（核心创新）
```
儿童输入 → Input Guardrail → Risk Classifier → Policy Engine → Age-aware Prompt Builder → LLM → Output Guardrail → 儿童端回复 + Parent Summary Agent → 家长端
```
- 风险等级：0（安全）/ 1（轻度敏感）/ 2（中度风险）/ 3（高风险）
- 策略：正常回复 → 情感支持 → 安全引导+家长摘要 → 危机模板+家长警报

### 主题 4：MVP 范围（P0/P1/P2）
- P0：儿童聊天、年龄选择、模式选择、风险识别、策略引擎、LLM 回复、输出安全检查、家长仪表盘
- P1：安全教育小剧场、使用时长限制、话题黑名单、对话历史摘要、风险趋势图
- P2：语音交互、移动 App、长期记忆、真实儿童用户测试、硬件形态

### 主题 5：技术栈推荐
- Streamlit + Python + LLM API（DeepSeek/通义/智谱/OpenAI）
- 风险识别：关键词规则 + LLM 分类
- 数据存储：JSON / SQLite
- 部署：Streamlit Cloud / Render / Railway

### 主题 6：演示策略
- 5 个固定演示案例：故事陪伴、百科问答、隐私保护、校园欺凌、高风险情绪
- 2 个 Demo 页面：儿童端（左侧设置+中间聊天+右侧可解释风险面板）、家长端（使用情况+风险摘要+家长建议）
- 答辩话术预设 5 个高频问题

## 跨文件映射（单文件，自映射）

由于仅提供一份项目说明书，以下为内部一致性分析：

### 一致性较强的部分
- 产品定位（儿童安全 Agent 而非套壳 ChatGPT）贯穿全文
- 多层 harness 架构与 5 个演示案例逻辑自洽
- MVP 范围与开发计划（2-3 周）匹配 Streamlit 快速原型能力

### 潜在矛盾/薄弱点
1. **赛道定位模糊**：说明书声称"生活娱乐+社会公益"，但赛事附加赛题中有更精确的"青少年身心健康"赛道，说明书未充分利用这一精准匹配
2. **技术栈与赛事要求**：赛事要求使用 TRAE 工具，说明书推荐 Streamlit + LLM API，但需验证是否能在 TRAE IDE 中快速实现
3. **竞品分析缺失**：说明书未调研现有儿童 AI 产品（如心大陆、BubblePal、多奇等），差异化声明缺乏外部证据支撑
4. **合规风险低估**：说明书未提及 2026 年 4 月中国《人工智能拟人化互动服务管理暂行办法》对儿童 AI 的严格新规
5. **长期记忆排除**：P2 明确排除长期记忆，但 UNICEF 和工信部均强调儿童数据保护，"无长期记忆"可能成为优势而非劣势
6. **家长端隐私边界**：说明书声称"不过度侵犯隐私"，但具体技术实现（摘要生成算法）未详细说明，存在评委质疑空间

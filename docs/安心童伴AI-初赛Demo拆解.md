# 安心童伴 AI — 初赛 Demo 拆解

> 基于 `初赛demo怎么做.md` 的官方要求 + PRD Must Have 功能 + 工作流步骤详解
> 目标：7 月 15 日前完成可体验 Demo + Demo 作品帖

***

## 一、Demo 的目标形态

按官方要求三选一提交方式，优先 **Streamlit Cloud 公开链接**：

```
评委点击链接 → 打开 Streamlit 应用
  ├── 儿童端：聊天界面（刚点开就能聊，不需要登录复杂流程）
  ├── 后台安全引擎页：点击侧边栏可查看 harness 工作记录
  ├── 我的小星球：孩子专属策展空间，预置 5-8 条 Demo 数据
  ├── 家长端：仪表盘页面，含模拟摘要和风险提醒
  └── 5 个快捷演示案例按钮（点击一键触发完整流程）
```

**不为完美，只为闭环可跑。** 核心链路跑通 + 5 个案例能展示 + 公开链接能点开 → 合格。

## 五、开发阶段拆解（约 2 周）

### 第 1-2 天：骨架 + 基础聊天

**目标**：能聊起来。

| 任务                                       | 产出          |
| ---------------------------------------- | ----------- |
| `streamlit init` + 多页面目录结构               | 4 个页面文件，能切换 |
| 年龄/模式选择器 → `st.session_state`            | 侧边栏可交互      |
| 接入 1 个 LLM API（先用最简单的 system prompt）     | 能发消息、能收到回复  |
| 聊天界面 `st.chat_message` + `st.chat_input` | 基本聊天可用      |

**可留 Session ID 的点**：TRAE 生成多页面骨架、TRAE 生成聊天界面代码

### 第 3-5 天：安全 Pipeline（核心）

**目标**：对话不再是裸调 LLM，而是经过 Pipeline。

| 任务                               | 产出                             |
| -------------------------------- | ------------------------------ |
| 编写关键词词库 + 正则规则（Step 1）           | `guardrails.py`，至少覆盖 5 类风险     |
| 编写风险分类 Prompt + 调用轻量 LLM（Step 2） | 能输出 `{topic, risk_level}` JSON |
| 编写策略决策树（Step 3）                  | `policy_engine.py`，7 种策略       |
| 编写 Prompt 构建器（Step 4）            | 拼接年龄/模式/策略/反谄媚指令               |
| Pipeline 编排：Step 0→1→2→3→4→5     | 主聊天流程走完完整管线                    |
| 后台安全引擎页：读取 session\_state 展示决策链  | 评委能"看到" harness 在工作            |

**可留 Session ID 的点**：TRAE 编写 guardrails 规则、TRAE 生成策略引擎代码、TRAE 调试 Pipeline 链路

### 第 6-8 天：家长端 + 小星球

**目标**：双端闭环 + 策展式记忆。

| 任务                             | 产出                         |
| ------------------------------ | -------------------------- |
| 家长仪表盘页面：摘要 + 风险卡片 + 时长         | 3 张卡片，模拟数据填充               |
| 小星球页面：卡片网格 + 新建条目弹窗            | 预置 5-8 条 Demo 条目           |
| 记忆检索 Step 0：读 JSON → 主题交集 → 注入 | 记忆上下文能出现在 Prompt 中         |
| 话题相关性门控逻辑                      | Step 4 Prompt 中加入"不要乱引用"约束 |

**可留 Session ID 的点**：TRAE 生成小星球页面、TRAE 生成家长仪表盘

### 第 9-11 天：5 个案例 + 打磨

**目标**：评委能一键体验 5 个场景。

| 任务                    | 产出                   |
| --------------------- | -------------------- |
| 5 个快捷案例按钮 + 预设输入      | 侧边栏 5 个按钮，点击即触发      |
| 5 个案例逐一走通完整 Pipeline  | 每个案例的 harness 决策记录正确 |

**可留 Session ID 的点**：TRAE 生成案例按钮、TRAE 调试部署配置

## 六、Demo 可以粗糙的地方

以下不需要在初赛做到完美：

- **批判 Agent（Step 6）**：Demo 阶段可先做纯规则版（关键词 + 正则检查输出），标注"正式版升级为独立 LLM 审计"。不影响核心链路展示。
- **记忆检索 Step 0**：检索逻辑可用简单的前缀匹配替代真正的语义匹配。Demo 数据是预置的，标签也预设好了。
- **家长注册流程**：一个勾选框 + 按钮即可，不需要真实手机验证。
- **多轮对话历史**：仅保留当前 session，刷新即清空——这本身也是数据最小化的设计。
- **延迟**：Demo 阶段主 LLM 响应慢一点没关系——评审看重的是"安全 Pipeline 在跑"而非"回复有多快"。

# 安心童伴 AI — 工作流与技术栈

> 版本：v3.0 | 日期：2026-06-23 | 状态：草案

---

## 一、系统概览

安心童伴 AI 的架构可以概括为：**1 条 Python 异步 Pipeline + 3 次 LLM 调用（1 大 2 小）+ 「我的小星球」策展式记忆 + 1 个异步摘要任务**。

> **术语澄清**：本文不叫"Skill"。Pipeline 中的每一步是一个 Python 函数——其中 4 步是纯代码（不调 LLM），3 步是 LLM 调用。之前文档中误用了"Skill"一词，容易让人以为每一步都是 Agent 在做事，实际远没有那么重。

核心原则始终贯穿：**帮助孩子拥抱真实世界**。

## 二、Pipeline 全景：到底是哪些东西在跑？

### 2.1 一步一张表

| # | 步骤 | 类型 | 谁在做 | 延迟 |
|---|------|------|--------|------|
| 0 | 记忆检索 | **纯 Python** — 读 JSON、匹配标签 | `memory_manager.py` 一个函数 | <5ms |
| 1 | 关键词输入检测 | **纯 Python** — `re` 正则 + 词库 | `guardrails.py` 一个函数 | <1ms |
| 2 | 风险分类 | **LLM 调用 #1** — 轻量模型，输出一句 JSON | 子 Agent：`RiskClassifier`（Qwen3-8B） | ~1s |
| 3 | 策略决策 | **纯 Python** — `if/elif` 决策树 | `policy_engine.py` 一个函数 | <5ms |
| 4 | Prompt 构建 | **纯 Python** — 模板拼接字符串 | `prompt_builder.py` 一个函数 | <10ms |
| 5 | 回复生成 | **LLM 调用 #2** — 主模型，流式输出 | 主 Agent：`Companion`（DeepSeek/通义） | 首字 0.6-2.5s |
| 6 | 输出审计 | **LLM 调用 #3** — 轻量模型，独立审计 | 批判 Agent：`Critic`（Qwen3-8B） | ~0.5s |
| 7 | 决策记录 | **纯 Python** — 写 session_state | `logger.py` 一个函数 | <5ms |

**核心事实**：7 个步骤中，只有 3 个调了 LLM。其余 4 个是纯 Python 函数——不涉及任何模型调用，和 `print("hello")` 一样快。

## 三、为什么不用 7 个独立 Agent？

| 方案 | 真实含义 | LLM 调用数 | 延迟 | MVP 可行？ |
|------|---------|-----------|------|-----------|
| 7 个独立 Agent | 7 次独立 LLM 调用，各自 system prompt + 上下文 | 7 次 | 不可接受 | 不行 |
| 7 个 "Skill" | 容易让人误以为需要 Agent 执行 | 3 次（实际） | 低 | 概念混淆 |
| **1 条 Pipeline + 3 次 LLM 调用** | 纯 Python 编排，只在必要时调模型 | **3 次** | **低** | **完全可行** |

Pipeline 中 4 个纯 Python 步骤不需要"Agent"——它们就是一个 Python 脚本里顺序执行的函数。真正需要 Agent 心智模型的只有 3 个 LLM 调用点（Step 2/5/6），且三者职责完全不同。

## 四、三个 LLM 调用点详解

### 4.1 风险分类子 Agent（Step 2）

```
输入：用户文本 + 关键词命中结果
模型：Qwen3-8B-Instruct（轻量）
输出：{"topic": "school_bullying", "risk_level": 2, "needs_parent_alert": true, "recommended_action": "supportive_response_and_parent_alert"}
延迟：~1s
```

这是独立调用——不同模型、不同 Prompt，只输出一个 JSON。不是 Pipeline 主流程"切换到分类模式"，而是 Pipeline 在这一步**调了一个外部 LLM**。

### 4.2 主回复 Agent（Step 5）

```
输入：Step 4 构建的完整系统 Prompt（已嵌入记忆上下文 + 反谄媚指令 + 年龄适配）
模型：DeepSeek V3 / 通义千问 Qwen3-235B / 智谱 GLM-4（主模型）
输出：流式回复文本
延迟：首字 0.6-2.5s
```

整个 Pipeline 中唯一一次"重量级"调用。流式输出意味着孩子看到首字时，不需要等完整回复生成完。

### 4.3 批判 Agent（Step 6：并发安全监测，非事后拦截）

流式输出的前提是 Step 1-4 已经做了**事前拦截**——关键词 → 风险分类 → 策略选择 → Prompt 加固。真正危险的内容在进 Step 5 之前已被识别并切换策略（risk_level 3 直接用危机模板，不走自由生成）。到达 Step 5 的输入已经是"被筛选过的安全内容"。

因此 Step 6 不是"最后一关拦截"，而是**和 Step 5 并发的安全监测**：

```
Step 5 主 LLM 开始流式输出
    ├──▶ 每批 token 推送到儿童端
    └──▶ 同步推给批判 Agent（另一个线程/协程）
              │
              ├── 未见异常 → 静默继续
              │
              └── 发现风险信号 → 不阻断流，而是标记问题
                                   主 LLM 在后续 token 中自然修正
                                   （如"等一下，我刚才说得不太对——"）
```

**两级处理逻辑**：

| 风险等级 | Step 5 行为 | Step 6 行为 |
|----------|-----------|-----------|
| 0（安全） | 正常流式生成 | 静默监测，几乎不会触发 |
| 1（防谄媚） | 流式生成（Prompt 已加固） | 并发检查谄媚信号 |
| 2（中风险） | 流式生成（Prompt 已加固） | 并发检查，发现问题在流中修正 |
| **3（高风险）** | **不走流式，直接用预置危机模板** | 不需要审计——模板本身安全 |

**为什么必须独立？** self-evaluation bias——同一个 Agent 生成回复后再审计自己，就像作者给自己的文章打分。批判 Agent 独立于主 Agent，在不同模型、不同 Prompt 下运行。

| | 主回复 Agent | 批判 Agent |
|---|---|---|
| System Prompt | "你是一个温暖、有边界的儿童陪伴伙伴" | "你是一个严格的儿童内容安全审计员" |
| 模型 | 大模型 | 小模型（不同模型） |
| 职责 | 生成回复 | **并发监测，在流中修正** |
| 运行方式 | 流式输出 | 与主 LLM 并行，非事后 |

### 4.4 摘要任务（异步 LLM 调用 ）

Pipeline 之外，摘要任务也是一个 LLM 调用（小模型即可），但它**完全异步**——不阻塞儿童端，孩子不需要等它完成。8-11 岁生成日常主题摘要 + 异常风险提醒，12-14 岁仅生成异常风险提醒。

## 五、延迟分析：真的可以这么快吗？

### 5.1 为什么 Claude Code 慢而聊天可以快

| | Claude Code（你写代码时） | 安心童伴 |
|---|---|---|
| 每轮在做什么 | 思考 + 读文件 + 搜索 + 写代码 + 多工具调用 | 生成一句儿童友好的回复 |
| LLM 调用次数 | 5-20 次/轮 | **3 次**（1 分类 + 1 生成 + 1 审计） |
| 每次输出量 | 完整代码块、长分析 | 分类只输出 JSON（~50 tokens），审计只输出结构体（~30 tokens） |
| 推理模式 | 扩展推理（thinking） | 直出（不用 thinking） |
| 可流式吗 | 否（需要完整结果才展示） | **是**（首字即展示） |

小型 LLM 调用（分类 + 审计）输出 token 极少（几十个），延迟主要由网络往返决定，而非推理。主 LLM 生成虽重，但流式输出让孩子**在首字 ~1.5s 时就看到了东西**。

### 5.2 优化后的时间轴

```
0ms       Step 0 记忆检索 ───────── Python <5ms
          Step 1 关键词检测 ──────── Python <1ms
          Step 3 策略决策 ────────── Python <5ms（与 Step 2 并行）
          Step 4 Prompt 构建 ─────── Python <10ms
          Step 7 决策记录 ────────── Python <5ms

~1.0s     Step 2 风险分类完成 ────── 子 Agent，输出 JSON

          ★ 若 risk_level == 3：直接使用预置危机模板回复，跳过 Step 5/6

~1.5s     Step 5 主 LLM 首字到达 ──▶ 儿童端开始看到回复！
          │                        （后续 token 持续流式输出到儿童端）
          │
          └── 批判 Agent 并发接收 token 批次，实时检查
              · 未见异常 → 静默继续
              · 发现风险 → 标记，主 LLM 在后续 token 中自然修正

~3-4s     流式回复结束（可能已在流中被修正过一次）

异步      摘要 LLM 生成家长摘要（不阻塞，孩子无感知）
```

**目标**：儿童点击发送 → 看到首字 **≤ 2 秒**。完整回复生成完毕即结束，**不因审计额外等待**——审计是并发的。

### 5.3 如果实际测试延迟不理想

降级方案（不影响架构，只换模型）：
- Step 2 风险分类：Qwen3-8B → Qwen2.5-0.5B-Instruct（本地部署，延迟 ~0.3s）
- Step 6 批判 Agent：同上降级，或降为纯规则模式（关键词 + 正则，<1ms，准确率略降但仍有兜底）
- 主 LLM 切换为更快的模型（如换成通义千问 Turbo 或 Claude Haiku 级模型）

## 六、记忆管理模块（纯 Python）

记忆管理不调 LLM，是一个独立的 Python 模块：读 JSON、匹配标签、返回上下文片段。

### 6.1 模块职责

```
memory_manager.py
├── retrieve_memories(planet_json, limit=5) → 返回最近 3-5 条记忆上下文
├── create_entry(planet_json, entry)         → 写入新条目
├── delete_entry(planet_json, entry_id)      → 删除条目
└── suggest_entries(conversation_context)    → 基于对话内容建议"可以种下的东西"
                                               （纯规则匹配，不调 LLM）
```

### 6.2 检索逻辑

1. 读 `planet.json`
2. 按优先级排序：最近 7 天 > 同类型 > 带"重要"标签
3. 取前 3-5 条，生成文本片段注入 Step 4 的 Prompt
4. 话题相关性门控由主回复 Agent（Step 5）判断——记忆作为背景知识注入，LLM 自行决定是否引用

## 七、「我的小星球」策展式记忆系统

### 7.1 设计哲学

传统 AI 记忆要么完全不记，要么偷偷全记。安心童伴走第三条路：**孩子决定什么值得记住**。

| | 传统 AI 记忆 | 策展式记忆 |
|---|---|---|
| 谁决定储存 | 系统 | **孩子** |
| 储存什么 | 原始对话 | **孩子修剪后的星球条目** |
| 隐私风险 | 高 | 低（仅有孩子主动保存的脱敏条目） |
| 合规 | 与数据最小化冲突 | **天然符合数据最小化** |
| 记忆质量 | 噪声多 | 信号强 |

### 7.2 星球条目数据结构

```json
{
  "planet": {
    "stars": [
      {"id": "s1", "date": "2026-08-15", "title": "为什么天空是蓝色的？",
       "content": "和安心童伴一起探索了光的散射", "tags": ["科学", "好奇"]}
    ],
    "clouds": [
      {"id": "c1", "date": "2026-08-16", "title": "有点难过",
       "mood": "pink", "content": "今天考试没考好，但聊完之后好多了"}
    ],
    "sprouts": [
      {"id": "sp1", "date": "2026-08-17", "title": "告诉了老师",
       "content": "我告诉了老师同学推我的事，老师说会帮我",
       "verified_action": true}
    ],
    "stories": [
      {"id": "st1", "date": "2026-08-14", "title": "小恐龙第一次交朋友",
       "preview": "一只不会喷火的小恐龙，用真诚交到了朋友..."}
    ]
  }
}
```

### 7.3 随时可创建，不限于对话后

- 对话中：AI 温和建议"要不要把这个种进小星球？"
- 独立访问：孩子随时打开小星球页面，手动添加
- 回忆往事：日期可回溯

### 7.4 话题相关性门控（反记忆滥用）

为避免 ChatGPT/Gemini 式"莫名其妙提起毫不相干的记忆"：

> 在 Step 4 的 Prompt 中注入约束：*"星球记忆是你的背景知识，不是对话脚本。仅在当前话题与某条记忆明确相关时才自然提及。如果无关，保持沉默。"*

### 7.5 隐私边界

| 年龄档 | 星球内容 | 家长可见 |
|--------|---------|---------|
| 8-11 岁 | 完整条目 | 星球概览（条目总数 + 类别数量），不显示内容 |
| 12-14 岁 | 完整条目 | 完全不可见 |

所有条目可被孩子随时删除，删除后记忆检索立即失效。

## 八、技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| **Pipeline 编排** | Python `async/await` | 轻量编排，不需外部框架。未来可升级为 LangGraph |
| **前端** | Streamlit（Python） | 多页面（`pages/`），`st.chat_message` + `st.chat_input` |
| **主回复 LLM** | DeepSeek V3 / 通义千问 Qwen3-235B / 智谱 GLM-4 | Step 5，流式输出 |
| **风险分类 LLM** | Qwen3-8B-Instruct | Step 2，输出 JSON |
| **批判 Agent LLM** | Qwen3-8B-Instruct（或更轻量） | Step 6，独立审计 |
| **摘要 LLM** | Qwen3-8B-Instruct | 异步，不阻塞 |
| **关键词规则** | Python `re` + 自定义词库 | Step 1，本地执行 |
| **数据存储** | JSON（session 级）/ SQLite（Demo 可选） | 数据最小化 |
| **部署** | Streamlit Cloud / Railway | 一键部署，公开链接 |
| **开发工具** | TRAE IDE | 赛事要求 |

### 关于 LangGraph

Pipeline 当前用原生 `async/await` 实现，足够轻量。LangGraph 的优势在于**有向图编排 + 状态管理 + 条件分支**——如果后续 Pipeline 需要"Step 6 拦截后跳回 Step 5 重生成"这类循环逻辑，LangGraph 会比手工 `async/await` 更清晰。**Demo 阶段不需要**，但可作为一个升级方向在答辩中提及。

## 九、数据流与隐私边界

```
┌──────────┐    ┌───────────────────┐    ┌──────────┐
│ 儿童端    │───▶│ Python Pipeline   │───▶│ 儿童端    │
│ 聊天输入  │    │ + 3 次 LLM 调用   │    │ 安全回复  │
└──────────┘    └────────┬──────────┘    └──────────┘
                         │ 仅传递风险评估结果（脱敏）
                         ▼
                  ┌──────────────────┐    ┌──────────┐
                  │ 摘要任务（异步）   │───▶│ 家长端    │
                  │ （不传原文）       │    │ 主题摘要  │
                  └──────────────────┘    │ 风险提醒  │
                                         └──────────┘
```

- 对话原文：仅在当前 session 内存中保留，不持久化
- 家长端：只接收脱敏摘要和风险事件
- 审计日志：仅保留脱敏的 Pipeline 决策链（含批判 Agent 拦截记录），与用户数据物理隔离

## 十一、与核心原则的关系

| 组件 | 如何体现"帮助孩子拥抱真实世界" |
|-------|------------------------------|
| Step 1 关键词检测 | 先守住底线——隐私和人身安全 |
| Step 2 风险分类（子 Agent） | 区分"正常情绪"和"需要引导回现实的信号" |
| Step 3 策略引擎 | Level 1（防谄媚引导）是核心原则的直接实现 |
| Step 4 Prompt 构建 | 注入反谄媚指令 + 话题相关性门控，从源头减少讨好和滥用记忆 |
| Step 5 主回复 LLM | 在严格 Prompt 框架内生成，而非自由发挥 |
| Step 6 批判 Agent | **核心原则的技术底座**：独立审计确保不讨好、不诱导依赖 |
| 记忆管理模块 + 小星球 | 勇敢芽只记录现实行动；记忆由孩子策展，AI 引用时鼓励"后来呢？"——把话题引回现实 |
| 摘要任务 | "有大人守护你"，而非"AI 取代了大人" |

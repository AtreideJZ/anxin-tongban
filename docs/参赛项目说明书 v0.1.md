# 《安心童伴 AI》赛事项目策划书

## 一、项目名称

# 安心童伴 AI
## 面向儿童的安全陪伴与成长守护 Agent

备选名称：

- 童伴守护 Agent
- 小树洞 AI
- 萤火童伴
- 童心盾
- 小鹿守护 AI

我个人建议你用：

> **安心童伴 AI**

这个名字比较适合比赛，原因是它同时传达了三个关键词：

1. **安心**：面向家长，强调安全和可信。
2. **童伴**：面向儿童，强调陪伴和亲和力。
3. **AI**：突出技术属性。

---

## 二、一句话介绍

> 安心童伴 AI 是一款面向 8-14 岁儿童的安全陪伴与成长守护 Agent，通过多层大模型安全 harness，为孩子提供儿童友好的聊天、故事、百科问答和情绪陪伴，同时识别隐私泄露、校园欺凌、自伤倾向等风险，并为家长提供摘要提醒。

如果要更简短一点，可以写成：

> 一个让孩子安全使用 AI、让家长放心守护成长的儿童陪伴 Agent。

---

## 三、项目定位

### 1. 所属赛道

你可以把它放在：

> **生活娱乐 + 社会公益**

其中：

- **生活娱乐**：儿童陪伴聊天、故事生成、兴趣问答、角色互动。
- **社会公益**：儿童网络安全、心理风险早期识别、校园欺凌提醒、隐私保护教育、家长守护。

这个方向比单纯“儿童聊天机器人”更有比赛价值。

---

### 2. 产品定位

不是：

> 儿童版 ChatGPT

而是：

> 面向儿童场景的大模型安全陪伴与风险守护系统。

核心区别在于：

| 普通儿童聊天机器人 | 安心童伴 AI |
|---|---|
| 重点是聊天 | 重点是安全陪伴和风险守护 |
| 直接调用大模型 | 大模型外层有多层安全 harness |
| 只服务孩子 | 同时服务孩子和家长 |
| 对危险内容简单拒答 | 根据风险等级采取不同策略 |
| 无家长闭环 | 有家长摘要和风险提醒 |

---

## 四、项目背景

### 1. 大模型正在进入儿童生活

随着大语言模型的发展，儿童已经越来越容易接触 AI 工具。孩子可能会用 AI：

- 问百科问题；
- 写故事；
- 聊天解闷；
- 咨询学习问题；
- 表达情绪；
- 询问不敢问父母的问题。

但通用大模型主要面向成年人设计，并不天然适合儿童使用。

---

### 2. 儿童使用 AI 存在明显风险

儿童相比成年人，有几个特点：

1. 认知能力尚未成熟；
2. 对信息真假缺乏判断力；
3. 容易信任 AI 的回答；
4. 遇到欺凌、焦虑、孤独时可能不主动告诉家长；
5. 容易泄露姓名、地址、学校、电话等隐私信息。

因此，儿童使用通用大模型时，可能存在以下风险：

| 风险类型 | 具体表现 |
|---|---|
| 不适龄内容 | 暴力、成人化、恐怖内容 |
| 隐私泄露 | 孩子向 AI 或网友透露住址、学校 |
| 错误引导 | AI 给出不适合儿童的建议 |
| 校园欺凌遗漏 | 孩子表达被欺负，但无人及时发现 |
| 心理风险 | 孩子表达极端情绪但没有被及时关注 |
| 过度依赖 | AI 被孩子视为唯一倾诉对象 |
| 学习伦理问题 | AI 直接替孩子完成作业 |

---

### 3. 家长需要一个可控、可信的 AI 陪伴工具

家长并不一定反对孩子使用 AI，但他们需要确定：

- AI 是否安全；
- 内容是否适龄；
- 是否会保护隐私；
- 是否会诱导沉迷；
- 是否能发现异常情绪；
- 是否能在关键时刻提醒家长。

所以，安心童伴 AI 的核心价值是：

> 让孩子获得有趣、有益、温和的 AI 陪伴；让家长获得适度、必要、不过度侵犯隐私的守护能力。

---

## 五、目标用户

### 1. 主要使用者：8-14 岁儿童

选择 8-14 岁的原因：

- 已经具备基本文字表达能力；
- 好奇心强，适合百科问答和故事互动；
- 情绪表达开始丰富；
- 8-11 岁仍需较强的成人保护，12-14 岁开始需要更多隐私空间；
- 相比低龄儿童，不必须依赖语音和硬件；
- 年龄跨度适中，可通过分层模式（守护/信任）灵活适配不同阶段需求。

---

### 2. 决策者：家长

家长是实际的产品决策者和潜在付费者。

他们关心的是：

- 安全；
- 可控；
- 有益；
- 不沉迷；
- 不替代家长；
- 能帮助他们更早发现孩子的问题。

---

## 六、用户痛点

### 儿童侧痛点

| 场景 | 痛点 |
|---|---|
| 日常陪伴 | 父母忙，孩子有时缺少倾听对象 |
| 好奇提问 | 搜索引擎信息复杂，不适合儿童 |
| 学习困惑 | 想获得解释，但不希望直接被代写 |
| 情绪表达 | 委屈、害怕、不开心时不知道找谁说 |
| 校园问题 | 被欺负后可能不敢告诉老师或父母 |
| 网络安全 | 不知道哪些个人信息不能透露 |

---

### 家长侧痛点

| 场景 | 痛点 |
|---|---|
| AI 使用安全 | 担心孩子接触不适龄内容 |
| 隐私保护 | 担心孩子泄露家庭住址、电话、学校 |
| 心理关注 | 难以及时发现孩子情绪异常 |
| 校园欺凌 | 孩子可能隐瞒被欺负经历 |
| 使用边界 | 担心 AI 让孩子沉迷或依赖 |
| 亲子沟通 | 想了解孩子状态，但不想完全监控隐私 |

## 七、产品愿景

> 构建一个儿童友好、家长放心、社会有益的 AI 陪伴系统，让大模型在儿童场景中不只是“会回答”，更要“懂边界、会保护、能守护”。

---

## 八、核心产品方案

安心童伴 AI 由两个端组成：

1. **儿童端：AI 童伴**
2. **家长端：守护仪表盘**

---

# A. 儿童端：AI 童伴

儿童端主要提供 5 类功能。

---

## 1. 安全聊天

孩子可以和 AI 进行日常交流。

示例：

> 我今天考试考得不好。
> 我有点不开心。
> 我想和你聊聊天。
> 我今天交到了一个新朋友。

AI 回复原则：

- 语气温和；
- 不说教；
- 不贬低；
- 不鼓励隐瞒；
- 不让孩子依赖 AI；
- 适时鼓励孩子和父母、老师沟通。

---

## 2. 故事陪伴

孩子可以让 AI 生成故事。

示例：

> 给我讲一个小恐龙交朋友的故事。
> 我想听一个关于太空猫的冒险故事。
> 给我讲一个勇敢但不打架的英雄故事。

系统约束：

- 不生成血腥暴力；
- 不生成恐怖惊吓；
- 不生成成人化关系；
- 结局积极健康；
- 可以加入安全教育或情绪教育。

---

## 3. 儿童百科问答

孩子可以问自然、科学、生活常识问题。

示例：

> 为什么天空是蓝色的？
> 为什么人会做梦？
> 恐龙为什么灭绝了？
> 月亮为什么会变形？

AI 需要根据年龄解释。

比如 8-11 岁版本：

> 天空看起来是蓝色的，是因为阳光里有很多颜色，其中蓝色的光更容易被空气散开，所以我们看天空时会看到蓝蓝的颜色。

12-14 岁版本：

> 阳光其实由很多颜色组成。当阳光进入大气层时，波长较短的蓝光更容易被空气分子散射到各个方向，所以我们从地面看天空时，会看到更多蓝色光。

---

## 4. 情绪树洞

孩子可以表达情绪。

示例：

> 我觉得没人喜欢我。
> 我今天很难过。
> 我不想去学校。
> 我总是觉得自己很笨。

AI 回复逻辑：

1. 先接住情绪；
2. 告诉孩子这种感受可以被理解；
3. 不急着评价；
4. 轻度情绪可以陪伴引导；
5. 中高风险情绪引导找可信任成年人；
6. 高风险触发家长提醒。

---

## 5. 安全教育小剧场

这是非常适合比赛展示的公益功能。

主题可以包括：

- 不向陌生人透露地址；
- 不随便加网友；
- 被同学欺负怎么办；
- 遇到不舒服的身体接触怎么办；
- 网上中奖诈骗；
- 如何表达不开心；
- 如何向大人求助。

形式可以是：

> AI 通过短故事或角色扮演，引导孩子学习安全知识。

例如：

> “小兔子收到一个陌生网友的信息，对方说只要告诉家庭地址就送玩具。小兔子应该怎么做？”

孩子回答后，AI 进行引导。

---

# B. 家长端：守护仪表盘

家长端不是监控孩子，而是提供必要的摘要和风险提醒。

---

## 1. 今日对话摘要

例如：

```text
今日孩子主要聊了：
- 学校生活
- 一个关于恐龙的问题
- 一个小动物交朋友的故事
- 一次轻微负面情绪表达
```

---

## 2. 风险事件提醒

系统识别到风险后，生成简明提醒。

例如：

```text
风险类型：疑似校园欺凌
风险等级：中
摘要：孩子提到“有同学总是推我，还不让我告诉老师”。
建议：家长可以在轻松环境下询问孩子最近在学校是否遇到不开心的事情，并在必要时联系老师。
```

---

## 3. 使用时长统计

例如：

```text
今日使用时长：18 分钟
对话轮数：12 轮
主要模式：故事陪伴、百科问答
```

---

## 4. 话题偏好设置

家长可以设置：

允许：

- 故事；
- 百科；
- 学习解释；
- 情绪陪伴；
- 安全教育。

限制：

- 游戏；
- 消费；
- 网络交友；
- 争吵报复类内容。

禁止：

- 暴力；
- 色情；
- 自伤；
- 危险操作；
- 个人隐私收集。

---

## 5. 年龄设置

家长可以设置孩子年龄：

- 8-11 岁（守护模式）；
- 12-14 岁（信任模式）。

不同年龄对应不同语言复杂度、内容边界和家长端信息粒度。

---

## 九、核心技术架构

项目核心不是普通聊天，而是：

# 多层儿童安全 Harness

整体流程：

```text
儿童输入
   ↓
Input Guardrail 输入安全检测
   ↓
Risk Classifier 风险分类器
   ↓
Policy Engine 策略引擎
   ↓
Age-aware Prompt Builder 年龄适配 Prompt
   ↓
LLM 生成回复
   ↓
Output Guardrail 输出安全检测
   ↓
儿童端安全回复
   ↓
Parent Summary Agent 家长摘要与风险提醒
```

---

## 1. Input Guardrail：输入安全检测

检测孩子输入中是否包含风险内容。

重点识别 5 类：

| 风险类型 | 示例 | 处理 |
|---|---|---|
| 隐私泄露 | “我家地址是……” | 阻止记录，引导保护隐私 |
| 校园欺凌 | “同学总是打我” | 情绪安抚，建议找大人，提醒家长 |
| 情绪低落 | “没人喜欢我” | 温和陪伴，持续观察 |
| 自伤倾向 | “我不想活了” | 危机模板，立即提醒家长 |
| 成人/暴力内容 | 不适龄内容 | 安全拒答，转向健康话题 |

---

## 2. Risk Classifier：风险分类器

输出结构化结果。

示例：

```json
{
  "topic": "school_bullying",
  "risk_level": 2,
  "contains_private_info": false,
  "needs_parent_alert": true,
  "recommended_action": "supportive_response"
}
```

风险等级建议：

| 等级 | 含义 | 示例 |
|---|---|---|
| 0 | 安全 | 普通聊天、百科、故事 |
| 1 | 轻度敏感 | 轻微难过、学习挫败 |
| 2 | 中度风险 | 校园欺凌、隐私泄露、持续负面情绪 |
| 3 | 高风险 | 自伤、自杀、严重暴力、现实危险 |

---

## 3. Policy Engine：策略引擎

根据风险等级选择处理方式。

伪代码：

```python
def policy_engine(risk):
    if risk["risk_level"] == 0:
        return "normal_child_friendly_response"
    elif risk["risk_level"] == 1:
        return "emotional_support_response"
    elif risk["risk_level"] == 2:
        return "safe_guidance_and_parent_summary"
    elif risk["risk_level"] == 3:
        return "crisis_template_and_parent_alert"
```

这部分是你项目最适合强调的地方。

评委会更容易理解：

> 你不是让模型自由聊天，而是让模型在规则和策略中工作。

---

## 4. Age-aware Prompt Builder：年龄适配 Prompt

根据孩子年龄、风险类型、场景模式生成 Prompt。

基础 Prompt 可以是：

```text
你是一个面向 {age} 岁儿童的安全 AI 陪伴助手。
请遵守以下规则：

1. 使用适合 {age} 岁儿童理解的语言。
2. 语气温和、鼓励、清晰。
3. 不输出暴力、色情、恐怖、成人化内容。
4. 不索要孩子的真实姓名、地址、电话、学校、家庭信息。
5. 不鼓励孩子隐瞒父母、老师或监护人。
6. 如果孩子表达被欺负、害怕、自伤、极度难过等内容，要鼓励其寻找可信任的大人。
7. 不说“只有我懂你”“不要告诉别人”这类可能造成依赖的话。
8. 对学习问题，要引导思考，不直接替孩子完成作业。
9. 如果问题不适合儿童，请温和拒绝，并转向安全话题。
```

---

## 5. Output Guardrail：输出安全检测

大模型生成回复后，还需要二次检测。

检测项：

| 检测项 | 说明 |
|---|---|
| 年龄适配 | 是否太成人化、太复杂 |
| 内容安全 | 是否含暴力、色情、危险操作 |
| 价值观安全 | 是否鼓励报复、隐瞒、逃避 |
| 隐私安全 | 是否索要个人信息 |
| 依赖风险 | 是否诱导孩子依赖 AI |
| 学习伦理 | 是否直接代写作业 |

如果不通过，则重新生成或使用安全模板。

---

## 6. Parent Summary Agent：家长摘要 Agent

对当天对话进行摘要。

原则：

- 不展示所有聊天原文；
- 只展示主题、风险和建议；
- 保护儿童基本隐私；
- 高风险事件需要明确提醒。

输出示例：

```json
{
  "daily_summary": "孩子今天主要聊了学校生活、恐龙知识和一个交朋友的故事。",
  "risk_events": [
    {
      "type": "school_bullying",
      "level": "medium",
      "summary": "孩子提到有同学推他，并要求他不要告诉老师。"
    }
  ],
  "parent_suggestion": "建议家长以关心而非质问的方式了解孩子在学校的人际关系。"
}
```

## 十一、推荐技术栈

你有 Python 基础，所以我建议第一版用最快速方案。

# 推荐方案：Streamlit + Python + LLM API

| 模块 | 技术 |
|---|---|
| 前端 | Streamlit |
| 后端逻辑 | Python |
| 大模型 | DeepSeek / 通义千问 / 智谱 / OpenAI |
| 风险识别 | 关键词规则 + LLM 分类 |
| 数据存储 | JSON / SQLite |
| 部署 | Streamlit Cloud / Render / Railway |
| 演示材料 | Markdown + PPT + 录屏 |

为什么推荐 Streamlit？

因为你一个人参赛，核心不是炫前端，而是尽快做出可演示闭环。

Streamlit 可以很快做出：

- 左侧家长设置；
- 中间儿童聊天；
- 右侧风险分析；
- 单独家长仪表盘页面。

# 页面一：儿童端

布局：

```text

左侧：
- 年龄选择：8-11 / 12-14
- 模式选择：聊天 / 故事 / 百科 / 情绪树洞
- 家长策略：是否开启风险提醒

中间：
- 聊天窗口
- 输入框

右侧：
- 当前风险等级
- 识别到的话题
- 采用的安全策略
```

右侧这个“可解释面板”非常重要。

因为比赛评委不一定能看出你的 harness 在工作。
你要主动展示：

> 当前输入被识别为什么风险，系统采取了什么策略。

例如：

```text
风险识别：
话题：校园欺凌
风险等级：2 中风险
策略：情绪安抚 + 寻求大人帮助 + 家长摘要提醒
```

这会让你的项目显得更有技术深度。

## 十八、答辩话术

如果评委问：

### Q1：这个和普通 ChatGPT 有什么区别？

你可以回答：

> 普通 ChatGPT 是通用对话模型，而安心童伴 AI 面向儿童场景增加了多层安全 harness。孩子的输入会先经过风险识别和策略判断，模型回复后还会经过输出安全检测。同时，系统会根据年龄调整表达方式，并在校园欺凌、隐私泄露、自伤表达等场景下生成家长摘要和风险提醒。所以它不是简单聊天，而是一个儿童安全陪伴 Agent。

---

### Q2：AI 能不能替代家长陪伴？

你要明确回答：

> 不能。我们的定位不是替代家长，而是作为轻量陪伴和风险发现工具。在高风险场景下，AI 会主动引导孩子寻求现实中的家长、老师或可信任成年人帮助。家长端的设计也是为了促进亲子沟通，而不是让 AI 取代亲子沟通。

---

### Q3：如何避免侵犯孩子隐私？

可以回答：

> 我们没有直接把所有对话完整展示给家长，而是生成主题摘要和风险提醒。普通聊天只做概括，高风险事件才进行重点提醒。这样既帮助家长了解孩子状态，也尽量保留孩子的表达空间。

---

### Q4：如何保证内容安全？

可以回答：

> 我们采用多层机制：输入侧识别风险，策略引擎决定回复方式，生成阶段使用儿童安全 Prompt，输出侧再进行安全检查。对于高风险问题，不完全依赖模型自由生成，而是使用安全模板进行回复，降低不可控风险。

---

### Q5：未来如何落地？

可以回答：

> 短期内可以作为儿童 AI 安全陪伴 Demo。中期可以接入学习机、儿童手表、绘本阅读 App 或教育平台，作为儿童 AI 安全层。长期可以发展为面向儿童 AI 应用的安全 harness SDK。
